# 카드뉴스 자동화 — 규칙 (단일 원본)

향/아로마·라이프스타일 개인 채널용 인스타그램 카드뉴스 제작 규칙.
`/cardnews` 스킬과 bg-image-generator 에이전트는 이 파일의 규칙을 따른다.

## §1 파이프라인

1. **소재 단계 (copy-ideator · Opus)** — 리서치 + 카드 구성안(표지/본문/CTA) + 카드별 핵심 메시지 + 무드 제안 → `output/{YYYY-MM-DD}-{주제}/plan.md` 초안.
1-2. **문구 단계 (text-designer · Sonnet)** — 구성안 기반으로 카드별 최종 카피 + 인스타 캡션 작성 → plan.md 완성 → 사용자 승인 대기. `자동` 모드에서는 승인 생략.
2. **이미지 단계 (병렬 · Haiku)** — 카드 수만큼 bg-image-generator 서브에이전트 동시 실행. 각자 Higgsfield `generate_image`로 4:5 배경 생성 → `bg/card-NN.png` 저장.
3. **조립 단계 (메인 스레드)** — Figma MCP로 1080×1350 프레임 생성, 배경 업로드, 텍스트 레이어 배치.
4. **품질 게이트 (§3)** — 5항목 검수. `자동` 모드에서 실패 카드는 최대 2회 재작업.
5. **전달** — PNG를 `final/`에 저장, Figma 링크 + PNG 전체 + 인스타 캡션(`caption.txt`) 전달.

## §2 포맷 규칙

- **규격**: 4:5 (1080×1350) 기본. 파생 요청 시 1:1 (1080×1080).
- **폰트**: Pretendard (Figma에 없으면 Noto Sans KR로 대체 — 2026-07-14 확인: Figma 클라우드 폰트에 Pretendard 미제공). 표지 제목 64~88px Bold, 본문 제목 48~52px Bold, 본문 텍스트 32~36px Regular, 페이지 표기 24~30px.
- **문체**: 표지 = 명사형 종결 ("깊은 밤, 라벤더의 온도"), 본문 = 해요체, CTA 카드 = 해요체 + 행동 유도 1문장.
- **카드 수**: 기본 7 (6~8 허용). 구조 = 표지 1 + 본문 4~6 + CTA 1.
- **무드**: 고정하지 않음. 카피 단계에서 세트별 무드를 제안하고 승인받는다. 승인된 무드는 `brand/style-presets.md`에 추가해 축적한다.
- **안전 영역**: 텍스트는 프레임 가장자리에서 60px 이상 안쪽. 이미지 주요 피사체 위에 텍스트 금지.
- **저장 경로**: `output/{YYYY-MM-DD}-{주제-슬러그}/` 아래 `plan.md`, `bg/`, `final/`, `caption.txt`.

## §3 품질 게이트

각 카드에 대해 아래 5항목을 검수한다. 하나라도 실패하면 해당 카드는 재작업 대상.

| 항목 | 기준 |
|---|---|
| 가독성 | 텍스트-배경 대비 충분. 부족하면 어두운 그라데이션 오버레이를 추가한다 |
| 안전 영역 | 텍스트가 가장자리 60px 이내 침범 금지, 주요 피사체 위에 걸림 금지 |
| 규격 | 1080×1350 정확, 텍스트 잘림 없음 |
| 카피 | 오탈자 없음, §2 문체 규칙 준수 |
| 이미지 결함 | AI 아티팩트(왜곡된 손, 뭉개진 글자 등) 없음 — 스크린샷으로 확인 |

재작업 규칙: `자동` 모드에서 카드당 최대 2회. 2회 초과 시 실패 카드 목록과 함께 부분 결과를 전달한다.

## §4 트리거

기본:

```
/cardnews 주제: {주제}
```

전체 옵션:

```
/cardnews 주제: {주제} 카드수: {6~8} 강조: {메시지} 참고: {URL} 자동
```

`자동` = 승인 생략 + 실패 카드 최대 2회 자동 재작업 + 최종 결과만 전달.

## §5 수정 지시 (제작 완료 후)

| 상황 | 지시 예시 | 동작 |
|---|---|---|
| 특정 카드 재생성 | "card-03 배경만 더 어두운 무드로 재생성해줘" | 해당 카드만 2~4단계 재실행 |
| 문체 일괄 변경 | "전체를 '~다'체로 바꿔줘" | Figma 텍스트 수정 → 재검수 → 재내보내기 |
| 규격 파생 | "1:1로도 내보내줘" | 프레임 복제 후 1080×1080 리레이아웃 |
| 스타일 변경 | "다음 세트는 미니멀 오브제 스타일로" | `brand/style-presets.md`에 기록, 다음 세트 기본값 |

## §6 오류 처리

- **Figma MCP 미인증**: 3단계 진입 전 감지. 인증 안내 후 중단하되 1·2단계 산출물(plan.md, bg/)은 보존.
- **Higgsfield 생성 실패**: 해당 카드만 재시도, 2회 실패 시 사용자에게 보고.
- **자동 모드 재작업 초과**: 실패 카드 목록과 함께 부분 결과 전달.

## §7 모델 배정

| 단계 | 담당 | 모델 | 이유 |
|---|---|---|---|
| 소재 리서치·구성안 | copy-ideator | Opus 4.8 | 소재 발굴·구조 설계는 가장 깊은 추론 필요 |
| 카드 문구·캡션 | text-designer | Sonnet 5 | 문체 규칙 내 카피라이팅, 품질·비용 균형 |
| 배경 이미지 생성 | bg-image-generator | Haiku 4.5 | 프롬프트 전달·다운로드 위주 기계적 작업 |
| 오케스트레이션·Figma 조립·품질 게이트 | 메인 스레드 | 세션 모델 | 승인 컨텍스트 유지 필요 |
