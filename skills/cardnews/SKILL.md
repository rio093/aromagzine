# 카드뉴스 제작 오케스트레이터

규칙의 단일 원본은 `~/cardnews/CLAUDE.md`다. 시작 전 반드시 읽고, 이 스킬 본문과 충돌하면 CLAUDE.md를 따른다.

## 입력 파싱

인자에서 추출: `주제`(필수), `카드수`(6~8, 기본 7), `강조`(선택), `참고`(URL/파일, 선택), `자동` 플래그.
주제 없으면 사용자에게 질문.

## 실행 순서

### 0. 준비

- `~/cardnews/CLAUDE.md`, `~/cardnews/brand/style-presets.md` 읽기.
- 세트 디렉토리 생성: `~/cardnews/output/{YYYY-MM-DD}-{주제-슬러그}/` + `bg/`, `final/`.
- Figma MCP 연결 확인 (도구 목록에 figma 도구 존재 여부). 미인증이면 지금 사용자에게 알리되, 1·2단계는 진행 가능하므로 중단하지 않는다.

### 1. 카피 단계 (메인 스레드)

- `참고` 자료가 있으면 읽는다 (WebFetch/Read).
- 작성: 카드별 구성안(표지 1 + 본문 N + CTA 1), 카드별 카피(§2 문체 규칙), 이번 세트 무드 제안(스타일 태그 3~5개 + 컬러 방향), 인스타 캡션 초안.
- `plan.md`로 저장 후 사용자에게 요약 제시 → 승인 대기.
- `자동` 모드: 승인 생략, 바로 2단계.
- 승인된 무드는 `brand/style-presets.md`에 한 줄 추가.

### 2. 이미지 단계 (병렬 서브에이전트)

- 카드마다 bg-image-generator 에이전트를 **한 메시지에 동시 디스패치**.
- 각 에이전트에 전달: 카드 번호, 이미지 프롬프트(무드 태그 + 카드 주제 반영, 영문 작성), 저장 경로 `bg/card-NN.png`, 규격 4:5.
- 프롬프트 공통 원칙: 사진 스타일(photographic), 텍스트 들어갈 여백 확보(negative space), 글자·손 포함 금지.
- 실패 카드는 해당 카드만 재시도(최대 2회).

### 3. 조립 단계 (메인 스레드)

- Figma 도구는 ToolSearch로 로드. `use_figma` 호출 전 `/figma-use` 스킬 필수.
- 새 파일 생성 → 카드당 1080×1350 프레임 → 배경 이미지 업로드·배치 → 텍스트 레이어(Pretendard, §2 크기 규칙) → 가독성 부족 시 그라데이션 오버레이 삽입.
- Figma 미인증 상태면 여기서 중단하고 인증 안내. plan.md와 bg/는 보존됨을 알린다.

### 4. 품질 게이트

- 카드별 스크린샷을 받아 §3의 5항목 검수.
- 실패 카드: 원인별 재작업 — 카피 문제면 1단계 일부, 이미지 문제면 2단계 해당 카드, 레이아웃 문제면 3단계 해당 프레임.
- `자동` 모드 재작업 한도: 카드당 2회.

### 5. 전달

- 각 프레임 PNG 내보내기 → `final/card-NN.png`.
- 캡션 확정 → `caption.txt`.
- 최종 보고: Figma 링크, PNG 전체(SendUserFile), 캡션, 품질 게이트 결과 요약(실패·재작업 이력 포함).

## 수정 지시 처리

제작 완료 후 같은 세션에서 §5(CLAUDE.md) 표의 지시가 오면 전체 파이프라인을 다시 돌리지 말고 해당 단계만 재실행한다.
